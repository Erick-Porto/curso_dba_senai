Níveis de Isolamento Padrão SQL (1/2)

O padrão SQL define quatro níveis de isolamento (a implementação exata pode variar um pouco entre SGBDs):

  1. READ UNCOMMITTED (Leitura Não Commitada)
    • É o nível de isolamento mais baixo.
    • Uma transação pode ler alterações feitas por outras transações que ainda não foram commitadas.
    • Anomalias Permitidas: Leitura Suja, Leitura Não Repetível, Leitura Fantasma.
    • Prós: Máxima concorrência, performance pode ser alta.
    • Contras: Risco muito alto de ler dados inconsistentes e "falsos". Raramente usado em aplicações críticas.
  2. READ COMMITTED (Leitura Commitada)
    • Garante que uma transação só leia dados que já foram commitados por outras transações.
    • Previne: Leitura Suja.
    • Anomalias Ainda Permitidas: Leitura Não Repetível, Leitura Fantasma.
    • É o nível de isolamento padrão em muitos SGBDs populares (PostgreSQL, SQL Server, Oracle).
    • Oferece um bom equilíbrio entre consistência e concorrência para muitas aplicações.
