Integridade de Dados: Garantindo Consistência

Assegura precisão, consistência e confiabilidade dos dados.

Tipos Principais de Restrições (Constraints):

  • Integridade de Entidade (PRIMARY KEY): Garante linhas únicas. PK não pode ser NULL.
  • Integridade Referencial (FOREIGN KEY): Garante relacionamentos consistentes. FK deve existir na PK referenciada ou ser NULL (se permitido).
    • Ações: ON DELETE CASCADE, SET NULL, RESTRICT...
  • Integridade de Domínio: Garante valores válidos nas colunas.
    • Tipos de Dados (INT, DATE...)
    • NOT NULL
    • UNIQUE
    • CHECK (condições)
    • DEFAULT (valor padrão)
