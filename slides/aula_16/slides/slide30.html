Caching e Resultados Materializados (Breve Visão)

Cache de Consultas do MySQL (Query Cache):

  • Histórico: Era uma funcionalidade que armazenava o texto da query e seu resultado.
  • Status Atual: Obsoleto desde MySQL 5.7.20 e REMOVIDO no MySQL 8.0.
  • Motivo: Apresentava problemas de escalabilidade e contenção de locks em ambientes com muitas escritas.

Outras Camadas de Cache Importantes:

  • Buffer Pool do InnoDB: CRUCIAL! Área de memória onde o InnoDB armazena dados e índices frequentemente acessados das tabelas. Reduz drasticamente o I/O de disco.
  • Cache de Aplicação: Implementado na sua aplicação (ex: Redis, Memcached) para armazenar resultados de queries frequentes ou objetos processados.
  • Cache de Sistema Operacional: O S.O. também faz cache de arquivos em disco.

Vistas Materializadas (Materialized Views):

  • Conceito: Uma tabela que armazena o resultado pré-calculado de uma consulta.
  • MySQL Standard: Não possui suporte nativo direto.
  • Alternativas:
    • Podem ser simuladas com tabelas de sumário atualizadas por triggers ou jobs agendados.
    • Alguns forks do MySQL (como MariaDB) ou SGBDs (como PostgreSQL) têm suporte nativo.
  • Vantagens: Leituras muito rápidas para queries complexas/agregadas.
  • Desvantagens: Dados podem ficar desatualizados até a próxima atualização; sobrecarga para manter a vista atualizada.
