Como Interpretar o EXPLAIN (Principais Colunas)

O output do EXPLAIN (tabular) contém várias colunas importantes:

Coluna Descrição
idOrdem de execução das operações.
select_typeTipo de SELECT (SIMPLE, SUBQUERY, UNION, etc.).
tableTabela referenciada.
typeMUITO IMPORTANTE! Tipo de acesso/join. Do melhor ao pior: system > const > eq_ref > ref > range > index > ALL (Full Table Scan - EVITAR!).
possible_keysÍndices que o MySQL poderia usar.
keyÍndice que o MySQL realmente usou. (NULL se nenhum)
key_lenTamanho da parte do índice usada.
rowsEstimativa de linhas a serem lidas para produzir o resultado.
filteredPercentual de linhas filtradas pela condição WHERE (após ler 'rows').
ExtraInformações adicionais: "Using where", "Using index", "Using filesort", "Using temporary".

O Visual Explain Plan no MySQL Workbench ajuda muito a entender essas relações!
