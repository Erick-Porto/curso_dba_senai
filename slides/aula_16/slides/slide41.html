Sumário da Aula - Pontos-Chave

Revisão dos aprendizados mais importantes de hoje:

  • O monitoramento de desempenho é crucial e deve ser uma prática proativa.
  • Identificar gargalos (bottlenecks) é fundamental: CPU, Memória, I/O, Rede, Queries.
  • O MySQL Workbench é uma ferramenta gráfica valiosa para monitoramento (Server Status) e análise de queries (Visual Explain).
  • Comandos como SHOW STATUS e o INFORMATION_SCHEMA são fontes ricas de informação interna do MySQL.
  • O comando EXPLAIN é seu melhor amigo para entender como o MySQL executa suas consultas.
  • Índices são poderosos para acelerar leituras, mas devem ser criados e gerenciados com sabedoria (impactam escritas).
  • Técnicas como evitar SELECT *, escrever cláusulas WHERE eficientes, otimizar JOINs e usar LIMIT são essenciais.
  • A otimização é um processo contínuo de análise, ajuste e aprendizado.
