Tipos de Tabelas Temporárias (Parte 1/2)

A nomenclatura e o comportamento exato podem variar entre SGBDs (SQL Server, Oracle, MySQL, PostgreSQL), mas os conceitos gerais são similares.

1. Temporary Tables (Padrão / Locais)

  • Visibilidade: Visíveis apenas durante a sessão que as criou. Nenhuma outra sessão pode vê-las ou acessá-las.
  • Nomenclatura (Exemplos Comuns):
    • SQL Server: #nome_tabela (local), ##nome_tabela (global temporária, mas ainda ligada à sessão criadora para descarte)
    • MySQL/PostgreSQL: CREATE TEMPORARY TABLE nome_tabela ...
  • Ciclo de Vida: Automaticamente descartadas (DROP) quando a sessão que as criou termina.
  • Isolamento: Se duas sessões diferentes criarem uma tabela temporária com o mesmo nome (ex: #MinhaTemp), elas são instâncias completamente distintas e isoladas. Uma não afeta a outra.
