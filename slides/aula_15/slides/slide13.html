Tipos de Tabelas Temporárias (Parte 2/2)

2. Global Temporary Tables (GTTs)

Comuns em SGBDs como Oracle e DB2.

  • Definição Persistente: A definição (estrutura) da tabela é persistente no banco de dados e visível para todas as sessões, como uma tabela permanente.
  • Dados Isolados por Sessão: Apesar da definição ser global, os dados inseridos na GTT são isolados por sessão (ou por transação, dependendo da configuração como ON COMMIT PRESERVE ROWS vs ON COMMIT DELETE ROWS no Oracle).
  • Exemplo (Oracle): 
    CREATE GLOBAL TEMPORARY TABLE gtt_dados_sessao (
    id NUMBER,
    descricao VARCHAR2(100)
) ON COMMIT PRESERVE ROWS; -- Dados persistem até o fim da sessão
-- ou ON COMMIT DELETE ROWS; -- Dados são limpos após cada COMMIT
  • Ponto Chave: Estrutura compartilhada, dados privados por sessão.

3. In-memory Temporary Tables

  • Alguns SGBDs oferecem otimizações para tabelas temporárias que residem primariamente em memória (ex: Tabelas Temporárias Otimizadas para Memória no SQL Server).
  • Benefício: Acesso extremamente rápido aos dados.
  • Cuidado: Consumo de memória RAM; não adequadas para volumes muito grandes de dados temporários.
