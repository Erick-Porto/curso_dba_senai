Sumário da Aula: Pontos-Chave

Parabéns por concluir esta aula sobre Tabelas Temporárias! Vamos revisar os pontos mais importantes:

  • Definição: Tabelas Temporárias são "rascunhos" da sessão, usadas para armazenar dados transitórios.
  • Escopo e Ciclo de Vida: Geralmente visíveis apenas na sessão criadora e destruídas automaticamente ao final desta.
  • Principais Usos:
    • Armazenar dados intermediários de consultas complexas.
    • Simplificar a lógica de consultas e relatórios.
    • Manipular grandes volumes de dados sem afetar tabelas principais diretamente.
  • Tipos: Locais (padrão), Globais (GTTs - ex: Oracle, com definição persistente mas dados por sessão) e In-memory (otimizadas).
  • Criação: CREATE TEMPORARY TABLE (ou #tabela no SQL Server). CTAS (CREATE TEMPORARY TABLE ... AS SELECT ...) é uma forma eficiente de criar e popular.
  • Manipulação: Comandos DML padrão (INSERT, SELECT, UPDATE, DELETE) dentro da sessão.
  • Boas Práticas: Use com moderação, nomeie bem (prefixos), evite conflitos de nome, crie índices se necessário para performance, e considere DROP explícito em scripts longos.
  • Cuidados: Atenção às permissões, impacto no tempdb/tablespace temporário, ausência de FKs diretas, e comportamento em replicação/clusters.

Dominar tabelas temporárias é uma habilidade essencial para um DBA eficiente!
