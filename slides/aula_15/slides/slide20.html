Utilização de Tabelas Temporárias

Uma vez criadas, as tabelas temporárias são manipuladas de forma muito similar às tabelas permanentes, dentro da sessão que as criou. Você pode usar os comandos DML (Data Manipulation Language) padrão:

1. Inserção de Dados (INSERT)

-- Inserir valores diretamente
INSERT INTO nome_tabela_temporaria (coluna1, coluna2)
VALUES (valor1, valor2);

-- Inserir dados de outra tabela
INSERT INTO nome_tabela_temporaria (coluna1, coluna2)
SELECT campo_origem1, campo_origem2
FROM outra_tabela
WHERE condicao;

2. Consulta de Dados (SELECT)

SELECT * FROM nome_tabela_temporaria WHERE condicao;

3. Atualização de Dados (UPDATE)

UPDATE nome_tabela_temporaria
SET coluna1 = novo_valor
WHERE condicao;

4. Exclusão de Dados (DELETE)

DELETE FROM nome_tabela_temporaria WHERE condicao;

5. Exclusão da Tabela (DROP TABLE - Opcional)

Embora sejam destruídas automaticamente ao fim da sessão, você pode removê-las explicitamente para liberar recursos mais cedo, especialmente em scripts longos:

DROP TABLE nome_tabela_temporaria;
-- No SQL Server: DROP TABLE #nome_tabela_temporaria;
