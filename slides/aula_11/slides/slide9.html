CREATE TABLE: Constraints Iniciais (As Regras Fundamentais)

Constraints (Restrições) são regras aplicadas às colunas ou à tabela para garantir a integridade e a validade dos dados.

As mais comuns definidas diretamente na coluna:

  • PRIMARY KEY (PK):
    • Identifica unicamente cada linha na tabela.
    • Só pode haver UMA por tabela (pode ser composta por múltiplas colunas).
    • Implica automaticamente NOT NULL e UNIQUE.
    • Ex: ClienteID INT PRIMARY KEY
  • NOT NULL (NN):
    • Impede que a coluna aceite valores NULL (vazios).
    • Garante que um campo obrigatório seja sempre preenchido.
    • Ex: Nome VARCHAR(100) NOT NULL
  • UNIQUE (UQ):
    • Garante que todos os valores na coluna (ou combinação de colunas) sejam diferentes entre si.
    • Permite valores NULL (geralmente apenas um, mas varia entre SGBDs).
    • Ex: Email VARCHAR(100) UNIQUE (não podem existir dois clientes com o mesmo email).
    • Ex: CONSTRAINT uk_pedido_produto UNIQUE (PedidoID, ProdutoID) (Constraint de tabela para chave única composta).
