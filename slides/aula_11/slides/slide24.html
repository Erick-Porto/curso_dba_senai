Views: Utilidade e Vantagens

Por que usar Views?

  • Abstração / Simplificação:
    • Escondem a complexidade de consultas com múltiplos JOINs, cálculos complexos ou formatações.
    • Usuários ou aplicações podem fazer um SELECT simples na view sem precisar conhecer a estrutura complexa das tabelas base.
  • Segurança:
    • Podem restringir o acesso a dados sensíveis, expondo apenas colunas ou linhas específicas permitidas para um grupo de usuários.
    • Ex: Uma view `vw_SalariosRH` pode mostrar nomes e salários, enquanto uma `vw_DadosPublicosFuncionarios` mostra apenas nome e departamento.
  • Consistência Lógica:
    • Fornecem uma visão padronizada e consistente dos dados, mesmo que as tabelas base mudem (desde que a view seja mantida).
    • Garante que cálculos ou lógicas de negócio complexas sejam aplicados de forma uniforme.
  • Compatibilidade Reversa (Legado):
    • Se você reestrutura tabelas base, pode criar views com a estrutura antiga para manter aplicações legadas funcionando temporariamente.
