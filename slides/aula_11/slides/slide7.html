CREATE TABLE: Tipos de Dados Comuns (1/2)

Escolher o tipo de dado correto para cada coluna é crucial para eficiência e integridade.

  • Inteiros:
    • INT ou INTEGER: Números inteiros padrão (geralmente 4 bytes). Ex: Idade, Quantidade, IDs.
    • SMALLINT: Inteiros menores (2 bytes). Ex: Contagem pequena.
    • BIGINT: Inteiros muito grandes (8 bytes). Ex: IDs em tabelas gigantes, contadores de alta volumetria.
  • Texto:
    • VARCHAR(n): String de texto de tamanho variável até um máximo de n caracteres. Ocupa espaço apenas pelo tamanho real + pequena sobrecarga. Mais comum para nomes, emails, títulos curtos.
    • CHAR(n): String de texto de tamanho fixo de n caracteres. Sempre ocupa n bytes (preenche com espaços se necessário). Útil para códigos de tamanho fixo (Ex: Siglas de Estado 'SP', 'RJ').
    • TEXT (e variações como MEDIUMTEXT, LONGTEXT): Para textos longos sem limite prático definido. Use com moderação, pois pode impactar performance. Ex: Descrições, Conteúdo de artigos.
  • Booleanos:
    • BOOLEAN (Padrão SQL) ou BOOL: Armazena TRUE ou FALSE (e às vezes NULL).
    • MySQL usa TINYINT(1) como sinônimo comum para booleanos (0 = falso, 1 = verdadeiro).
    • Ex: Colunas Ativo, Publicado, EmailVerificado.
