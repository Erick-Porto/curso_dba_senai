Índices: Quando Usar? (Onde eles brilham )

Índices são mais eficazes em colunas que são:

  • Frequentemente usadas na cláusula WHERE para filtrar um subconjunto relativamente pequeno de linhas.
    • Ex: WHERE ClienteID = ?, WHERE StatusPedido = 'Pendente', WHERE Email = ?
  • Usadas em condições de JOIN (as colunas de chave estrangeira são excelentes candidatas, embora alguns SGBDs possam criá-los automaticamente).
    • Ex: ON Pedidos.ClienteID = Clientes.ClienteID (indexar Pedidos.ClienteID).
  • Frequentemente usadas na cláusula ORDER BY para classificar os resultados.
    • Ex: ORDER BY DataCadastro DESC (indexar DataCadastro).
  • Colunas com alta cardinalidade (muitos valores distintos) geralmente se beneficiam mais de índices B-Tree padrão do que colunas com baixa cardinalidade (poucos valores distintos, como um campo 'Sexo' M/F).

O objetivo é sempre reduzir a quantidade de trabalho que o SGBD precisa fazer para encontrar os dados solicitados.
