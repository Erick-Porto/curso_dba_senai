CREATE TABLE: Tipos de Dados Comuns (2/2)

  • Data e Hora:
    • DATE: Armazena apenas a data (Ano, Mês, Dia). Ex: '2024-08-15'.
    • TIME: Armazena apenas a hora (Hora, Minuto, Segundo). Ex: '14:30:00'.
    • DATETIME (MySQL) / TIMESTAMP WITHOUT TIME ZONE (PostgreSQL): Armazena data e hora combinadas. Ex: '2024-08-15 10:00:00'.
    • TIMESTAMP (MySQL) / TIMESTAMP WITH TIME ZONE (PostgreSQL): Armazena data e hora, geralmente convertendo para/de UTC ou armazenando o fuso horário. Útil para aplicações globais.
  • Números Decimais Exatos:
    • DECIMAL(p, s) ou NUMERIC(p, s): Armazena números com precisão fixa. p é o número total de dígitos, s é o número de dígitos após a vírgula decimal. Ideal para valores monetários para evitar erros de arredondamento. Ex: DECIMAL(10, 2) para preços (até 99.999.999,99).
  • Números Decimais Aproximados (Ponto Flutuante):
    • FLOAT, DOUBLE PRECISION / REAL: Armazenam números com ponto flutuante. Mais rápidos para cálculos científicos, mas podem sofrer de imprecisão. Geralmente EVITADOS para dinheiro.
  • Outros: BLOB (dados binários), JSON, ENUM, Tipos Geométricos, etc. (variam muito entre SGBDs).

Escolher o tipo certo economiza espaço e previne erros!
