Views: Atualização (Cuidado!)

É possível executar comandos DML (INSERT, UPDATE, DELETE) diretamente em uma View?

Resposta: Depende!

  • Views Simples (Geralmente Atualizáveis):
    • Baseadas em uma única tabela base.
    • Não usam funções agregadas (COUNT, SUM, AVG...).
    • Não usam GROUP BY ou HAVING.
    • Não usam DISTINCT.
    • Todas as colunas NOT NULL da tabela base sem valor DEFAULT devem estar incluídas na view para permitir INSERT.

    Ex: UPDATE vw_DepartamentosAtivos SET Localizacao = 'Nova Sede' WHERE DepartamentoID = 1; (Provavelmente funcionaria).

  • Views Complexas (Geralmente NÃO Atualizáveis):
    • Baseadas em múltiplos JOINs.
    • Usam agregações, GROUP BY, DISTINCT, etc.

    Ex: Tentar inserir em uma view que junta Clientes e Pedidos é ambíguo e geralmente proibido.

Regra Geral: As regras exatas variam entre SGBDs. Na dúvida, ou para lógica complexa, é sempre mais seguro e claro realizar as operações DML diretamente nas tabelas base.
