CREATE TABLE: Exemplo Completo e Boas Práticas

Vamos criar uma tabela Departamentos aplicando o que vimos:


-- Exemplo para MySQL (AUTO_INCREMENT)
CREATE TABLE Departamentos (
    DepartamentoID INT AUTO_INCREMENT PRIMARY KEY, -- Chave Primária auto-gerada
    NomeDepto VARCHAR(80) NOT NULL UNIQUE,      -- Nome obrigatório e não pode repetir
    Sigla VARCHAR(10) UNIQUE,                     -- Sigla única, mas pode ser opcional (NULL)
    CentroCusto VARCHAR(20),                      -- Pode ser nulo
    DataCriacao DATE DEFAULT CURRENT_DATE        -- Data com valor padrão (data atual)
);

-- Exemplo para PostgreSQL (SERIAL)
CREATE TABLE Departamentos (
    DepartamentoID SERIAL PRIMARY KEY,            -- Chave Primária auto-gerada
    NomeDepto VARCHAR(80) NOT NULL UNIQUE,
    Sigla VARCHAR(10) UNIQUE,
    CentroCusto VARCHAR(20),
    DataCriacao DATE DEFAULT CURRENT_DATE
);

Boas Práticas Iniciais:

  • Nomenclatura Clara: Tabela no plural (`Departamentos`), colunas no singular ou descritivas (`NomeDepto`, `DataCriacao`). Use um padrão consistente (`snake_case` ou `CamelCase`). Use `ID` ou `Codigo` claramente na PK.
  • Tipos e Tamanhos Adequados: `VARCHAR(80)` é suficiente para nome de depto? `VARCHAR(10)` para sigla? Evite `VARCHAR(255)` genérico se não precisar.
  • Restrições Explícitas: Use NOT NULL onde for mandatório. Use UNIQUE onde a lógica de negócio exigir unicidade. Defina a PRIMARY KEY.
  • Valores Padrão (DEFAULT): Útil para colunas como datas de criação ou status iniciais.
