Exercício 3 (Avançado): Relacionamentos e Visões

Sua Missão: Estruturar categorias, relacioná-las aos produtos e criar uma visão útil.

Tarefas:

  1. Crie uma nova tabela chamada Categorias com:
    • CategoriaID: Inteiro, PK, auto-incrementada.
    • NomeCategoria: VARCHAR(50), obrigatório (NOT NULL), único (UNIQUE).
  2. Altere a tabela Produtos: adicione uma coluna CategoriaID do tipo Inteiro (INT), que pode ser nula inicialmente.
  3. Adicione uma FOREIGN KEY na tabela Produtos (na coluna CategoriaID) que referencie a coluna CategoriaID da tabela Categorias. Defina a regra ON DELETE SET NULL (se uma categoria for apagada, o CategoriaID nos produtos relacionados se tornará NULL). Dê um nome à constraint (ex: `fk_produto_categoria`).
  4. Insira pelo menos 3 categorias de exemplo na tabela Categorias (ex: 'Eletrônicos', 'Vestuário', 'Alimentos').
  5. Crie uma VIEW chamada vw_ProdutosComCategoria que mostre o ProdutoID, NomeProduto, PrecoCusto da tabela Produtos e o NomeCategoria da tabela Categorias (você precisará de um JOIN!).

Dica: Para a view, use um LEFT JOIN de Produtos para Categorias para garantir que produtos sem categoria (CategoriaID NULL) ainda apareçam na view, mas com NomeCategoria NULL. Teste sua view com SELECT * FROM vw_ProdutosComCategoria;.
