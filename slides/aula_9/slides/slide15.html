Combinando Tabelas - Por Que JOINS?

Bancos relacionais organizam dados em tabelas separadas para evitar redundância (Normalização).

Ex: Tabela Clientes (info do cliente), Tabela Pedidos (info do pedido, incluindo ID do cliente).

Problema: Como ver o nome do cliente junto com a data do seu pedido?

Solução: Usar JOIN para combinar linhas de tabelas diferentes com base em uma coluna de ligação (chave primária/estrangeira).

Analogia: Juntar peças de um quebra-cabeça que se encaixam (IDs correspondentes) para formar uma imagem maior (informação combinada).
