Comparativo: Relacional vs. NoSQL

Uma visão geral (lembre-se que NoSQL é uma categoria ampla):

Característica Relacional (SQL) NoSQL (Geral)
Modelo de Dados Tabelas (linhas/colunas) Variado (Chave-Valor, Documento, Coluna, Grafo)
Esquema Rígido, pré-definido (Schema-on-write) Flexível, dinâmico (Schema-on-read)
Escalabilidade Vertical (geralmente) Horizontal (geralmente)
Consistência (Transações) Forte (ACID) Eventual (BASE - geralmente)
Linguagem de Consulta SQL (padrão) Variada, específica do banco (APIs, dialetos)
Relacionamentos Explícitos (Chaves Estrangeiras) Implícitos ou foco principal (Grafos)
Casos de Uso Típicos Sistemas transacionais (Bancos, ERPs), dados estruturados Big Data, caches, tempo real, conteúdo variado, redes sociais

Importante: A escolha depende MUITO do problema a ser resolvido!
