Estratégias de Armazenamento de Backups

Onde e como você armazena seus backups é tão importante quanto fazê-los.

A Regra de Ouro: 3-2-1

3 Cópias
(Original + 2 Backups)

2 Mídias Diferentes
(Ex: Disco local + Nuvem)

1 Cópia Off-site
(Local geograficamente separado)

Opções Comuns de Armazenamento:

  • Disco Local (no mesmo servidor):
    • Prós: Rápido para backup e restore. Contras: ALTÍSSIMO RISCO! Se o servidor falhar (disco, placa-mãe, desastre), você perde o original E o backup. Use apenas como um local temporário antes de mover para outro lugar.
  • Outro Disco Local (no mesmo servidor, mas fisicamente separado):
    • Melhor que o anterior, mas ainda vulnerável a falhas do servidor ou desastres locais.
  • Servidor de Backup Dedicado na Rede Local (NAS, SAN, outro servidor):
    • Prós: Mais seguro, isola os backups do servidor de produção. Contras: Ainda vulnerável a desastres que afetem todo o local (incêndio, inundação).
  • Armazenamento em Nuvem (Cloud Storage):
    • Ex: AWS S3, Google Cloud Storage, Azure Blob Storage, Backblaze B2.
    • Prós: Excelente para cópia off-site, alta durabilidade, escalabilidade, geralmente custo-efetivo. Contras: Velocidade de upload/download pode ser um gargalo, custos de transferência e armazenamento a longo prazo.
  • Servidor FTP/SFTP Remoto:
    • Uma opção para transferir backups para um local off-site. SFTP é preferível por segurança.
  • Fitas Magnéticas (LTO):
    • Menos comum para recuperação rápida, mas ainda usada para arquivamento de longo prazo (arquivamento "cold storage") devido ao baixo custo por TB e durabilidade offline.

Política de Retenção de Backups:

Defina por quanto tempo você precisa manter os diferentes tipos de backups (diários, semanais, mensais, anuais). Isso depende dos seus RPOs, requisitos de conformidade legal/setorial, e necessidades de negócio para recuperação histórica.
