Ferramentas de Backup: mysqldump (MySQL)

mysqldump é um utilitário de linha de comando cliente que pode ser usado para realizar backups lógicos de bancos de dados MySQL. Ele produz um arquivo de script SQL que pode ser executado para recriar o banco de dados e/ou suas tabelas e dados.

mysqldump

Principais Usos do mysqldump:

  • Backups Completos ou Parciais:
    • Backup de todos os bancos de dados no servidor.
    • Backup de um ou mais bancos de dados específicos.
    • Backup de tabelas específicas dentro de um banco de dados.
  • Migração de Dados: Transferir dados entre servidores MySQL (mesma versão ou versões diferentes, com atenção a compatibilidade).
  • Criação de Cópias para Ambientes de Desenvolvimento/Teste: Fornecer uma cópia dos dados de produção para equipes de desenvolvimento.
  • Clonagem de Estrutura: Exportar apenas a estrutura das tabelas (sem dados) usando a opção --no-data.

Características Importantes:

  • Saída Padrão: Por padrão, envia a saída SQL para stdout (saída padrão), que pode ser redirecionada para um arquivo.
  • Consistência: Para tabelas InnoDB, a opção --single-transaction é crucial para obter um backup consistente sem bloquear as tabelas por longos períodos. Ele inicia uma transação e faz o dump dentro dessa transação.
  • Flexibilidade: Oferece uma vasta gama de opções para controlar o que é incluído no backup (dados, estrutura, routines, triggers, events, etc.).
