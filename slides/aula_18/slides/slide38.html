Sumário da Aula - Pontos-Chave

Revisão dos aprendizados mais importantes de hoje sobre backup e recuperação:

  • Backup é fundamental para proteger contra perda de dados devido a falhas, erros ou desastres. Não é opcional!
  • Entendemos os tipos de backup: Completo (tudo), Incremental (mudanças desde o último backup), Diferencial (mudanças desde o último completo).
  • Diferenciamos backups Lógicos (SQL, portáteis, ex: mysqldump) de Físicos (arquivos binários, rápidos, ex: XtraBackup).
  • Point-in-Time Recovery (PITR) permite restauração granular, mas requer backups base e logs de transação (binlogs no MySQL).
  • RPO (quanta perda de dados é aceitável) e RTO (quanto tempo para recuperar) são métricas de negócio que guiam a estratégia de backup.
  • mysqldump é a ferramenta padrão para backups lógicos no MySQL, com opções importantes como --single-transaction.
  • A automatização de backups com scripts (Bash, PowerShell) e agendadores (cron) é crucial para consistência e confiabilidade.
  • Siga boas práticas de armazenamento (Regra 3-2-1, off-site) e nomenclatura de arquivos.
  • TESTE SEUS BACKUPS REGULARMENTE! Um backup não testado é uma esperança, não uma garantia.
