Demonstração: Contextualizando com o Modelo de Dados (DER Chen)

Antes de mergulharmos nas estatísticas e planos de consultas específicas, é crucial entendermos a estrutura do nosso banco de dados de exemplo.

Vamos usar o MySQL Workbench para visualizar o modelo:

Passos da Demonstração:

  1. Conectar ao servidor MySQL que contém o banco de dados sakila.
  2. No menu "Database", selecionar "Reverse Engineer..." (ou Ctrl+R).
  3. Seguir os passos do assistente para se conectar e selecionar o schema sakila.
  4. Após a importação dos objetos, o Workbench exibirá o Diagrama Entidade-Relacionamento (EER Diagram, que pode ser visto como uma representação DER Chen).
  5. Navegar pelo diagrama:
    • Identificar tabelas principais: film, actor, customer, rental, inventory, store, address, city, country.
    • Observar as colunas chave (PKs, FKs) e os relacionamentos entre as tabelas.
    • Entender como as tabelas se conectam para responder a perguntas de negócio (ex: "Quais filmes um cliente alugou?").

Este entendimento da estrutura é fundamental para escrever JOINs corretos e interpretar como o otimizador acessará os dados.
