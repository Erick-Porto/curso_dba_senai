Sumário da Aula - Pontos-Chave

Revisão dos aprendizados essenciais de hoje sobre otimização de consultas:

  • O Otimizador de Consultas do MySQL é um componente complexo que usa Estatísticas e Índices disponíveis para gerar o Plano de Execução mais eficiente (menor custo estimado).
  • Estatísticas precisas e atualizadas são cruciais. Use ANALYZE TABLE após mudanças significativas nos dados ou schema.
  • O comando EXPLAIN (e EXPLAIN ANALYZE) é a ferramenta fundamental para entender como o MySQL planeja executar suas queries. O Visual Explain do Workbench facilita a interpretação.
  • Colunas importantes do EXPLAIN incluem type (método de acesso), key (índice usado), rows (linhas estimadas) e Extra (informações vitais como "Using index", "Using filesort").
  • Índices (B-Tree) aceleram buscas por igualdade, intervalo e prefixo, além de otimizar JOINs e ORDER BY. No entanto, eles têm um custo nas operações de escrita.
  • Em Índices Compostos, a ordem das colunas é vital. O MySQL utiliza o prefixo mais à esquerda do índice.
  • Covering Indexes (resultando em "Using index" no EXPLAIN) são altamente eficientes, pois a query é respondida apenas lendo o índice, sem acessar a tabela.
  • A otimização é um processo iterativo: Analisar Identificar Gargalos Propor Otimizações (Índices, Reescrever Query) Medir Impacto Repetir.
