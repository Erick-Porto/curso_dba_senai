Planos de Execução - Definição

O que é um Plano de Execução (Execution Plan)?

É o "mapa do tesouro" ou a "receita detalhada" que o Otimizador de Consultas do SGBD cria para executar uma determinada instrução SQL. Ele descreve, passo a passo, como o SGBD pretende:

  • Acessar cada tabela envolvida (ex: Full Table Scan, Index Scan).
  • A ordem em que as tabelas serão processadas (em caso de JOINs).
  • O método de junção (JOIN) a ser usado entre as tabelas (ex: Nested Loop, Hash Join).
  • Como os filtros (cláusula WHERE) serão aplicados.
  • Se e como operações de ordenação (ORDER BY) ou agrupamento (GROUP BY) serão realizadas.
  • Estimativas de custo e número de linhas para cada etapa.

O plano de execução é a SAÍDA do processo de otimização.

Como o Otimizador Escolhe entre Várias Estratégias (Planos)?

O otimizador é um programa complexo que considera muitas alternativas:

  1. Geração de Planos Candidatos: Para uma mesma query, podem existir dezenas, centenas ou até milhares de formas diferentes de executá-la. O otimizador explora um subconjunto dessas possibilidades.
  2. Estimativa de Custo: Para cada plano candidato, o otimizador calcula um "custo" estimado. Esse custo é uma combinação de fatores como:
    • Número estimado de operações de I/O (leitura de páginas do disco).
    • Uso estimado de CPU.
    • Uso de memória (para ordenações, tabelas temporárias, hash joins).

    Essas estimativas são fortemente baseadas nas estatísticas do banco de dados!

  3. Seleção do Plano "Mais Barato": O otimizador seleciona o plano candidato que possui o menor custo total estimado.

Importante: O plano escolhido é baseado em ESTIMATIVAS. Se as estatísticas estiverem incorretas, o "menor custo estimado" pode não ser o melhor plano na prática.

Analisar o plano de execução é como verificar a rota que o GPS escolheu antes de iniciar a viagem, para garantir que não há desvios estranhos!
