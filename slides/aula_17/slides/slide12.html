Estatísticas de Banco de Dados - Ferramentas no MySQL

O MySQL oferece comandos e tabelas de sistema para interagir com as estatísticas:

ANALYZE TABLE nome_da_tabela;

  • Principal comando para solicitar a coleta ou atualização das estatísticas de uma ou mais tabelas e seus índices.
  • Para tabelas InnoDB, ele analisa uma amostra das páginas e armazena as estatísticas calculadas nas tabelas persistentes do sistema (mysql.innodb_table_stats, mysql.innodb_index_stats).
  • Demonstração Rápida no Workbench: 
    ANALYZE TABLE sakila.film;

    Observe a mensagem de status retornada.

SHOW INDEX FROM nome_da_tabela;

  • Mostra informações detalhadas sobre todos os índices de uma tabela.
  • A coluna Cardinality é crucial aqui: é a estimativa do SGBD para o número de valores únicos no índice. Quanto maior a cardinalidade em relação ao total de linhas da tabela, mais seletivo é o índice.
  • Demonstração Rápida no Workbench: 
    SHOW INDEX FROM sakila.film;

    Analise a coluna `Cardinality` para diferentes índices. Compare com o total de linhas da tabela (pode ser visto em `SHOW TABLE STATUS LIKE 'film';` ou `INFORMATION_SCHEMA.TABLES`).

Tabelas do INFORMATION_SCHEMA:

  • INFORMATION_SCHEMA.STATISTICS: Contém informações sobre índices, similar ao `SHOW INDEX`.
  • INFORMATION_SCHEMA.TABLES: Colunas como TABLE_ROWS (estimativa de linhas) e DATA_LENGTH, INDEX_LENGTH.
  • No MySQL 8.0+, information_schema.column_statistics pode armazenar histogramas se coletados.

PostgreSQL (para comparação): ANALYZE nome_tabela; e a view pg_stats.
