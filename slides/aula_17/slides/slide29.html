Exercício 1: Análise Básica e Impacto de SELECT *

Objetivo: Familiarizar-se com o EXPLAIN e observar o impacto de selecionar colunas específicas.

Banco de Dados: sakila.

Consulta 1.1 (Todas as Colunas):


EXPLAIN SELECT * FROM sakila.actor WHERE last_name = 'CHASE';

Consulta 1.2 (Colunas Específicas):


EXPLAIN SELECT actor_id, first_name, last_name
FROM sakila.actor WHERE last_name = 'CHASE';

Tarefas:

  1. Execute o EXPLAIN para a Consulta 1.1.
    • Analise o resultado tabular e o Visual Explain no Workbench.
    • Observe: type, possible_keys, key (deve ser idx_actor_last_name), rows, Extra.
  2. Execute o EXPLAIN para a Consulta 1.2.
    • Compare o plano com o da Consulta 1.1.
    • Houve alguma mudança significativa no plano de acesso (type, key, rows)?
    • O campo Extra mudou? A Consulta 1.2 se tornou um "Covering Index" (mostrando Using index)? Por quê? (Dica: O índice padrão idx_actor_last_name no Sakila é apenas na coluna `last_name`).
  3. Discussão: Qual o principal benefício de usar SELECT colunas_especificas em vez de SELECT *, mesmo que o plano de acesso não mude drasticamente neste caso específico?
