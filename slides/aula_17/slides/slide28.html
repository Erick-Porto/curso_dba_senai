Hora 4: Prática de Otimização no MySQL Workbench

Chegou a hora de aplicar o conhecimento! Vamos usar o MySQL Workbench para analisar e otimizar consultas no banco de dados sakila.

Foco da Prática:

  • Utilizar o Visual Explain e o EXPLAIN tabular.
  • Identificar gargalos e acessos ineficientes (ex: Full Table Scans, filesorts).
  • Criar índices simples e compostos para melhorar o desempenho.
  • Observar e interpretar as mudanças nos planos de execução após as otimizações.
  • Analisar o potencial para Covering Indexes.

Abram seus MySQL Workbench e conectem-se ao servidor com o banco de dados sakila. Se não o tiverem, scripts de instalação do Sakila são facilmente encontrados online.
