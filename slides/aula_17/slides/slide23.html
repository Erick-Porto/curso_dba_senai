Índices: Tipos de Índices no MySQL

O MySQL suporta diversos tipos de índices para diferentes necessidades:

  • Índice Simples (Single-column Index):

    Criado sobre uma única coluna da tabela. Ex: CREATE INDEX idx_actor_lastname ON actor(last_name);

  • Índice Composto (Multi-column Index / Composite Index):

    Criado sobre duas ou mais colunas. Ex: CREATE INDEX idx_film_title_release ON film(title, release_year);
    A ordem das colunas no índice composto é FUNDAMENTAL! Um índice em (A, B) pode ser usado para buscas em A ou em (A,B), mas geralmente não para buscas apenas em B (a menos que seja um "index skip scan" em SGBDs mais avançados, menos comum no MySQL). O MySQL pode usar a parte mais à esquerda do índice (leftmost prefix).

  • Chave Primária (PRIMARY KEY):

    Um tipo especial de índice único que identifica unicamente cada linha. No InnoDB, a PK é um clustered index, o que significa que os dados da tabela são fisicamente armazenados na ordem da PK. Todas as outras chaves secundárias (índices não-PK) no InnoDB contêm a PK para referenciar a linha.

  • Índice Único (UNIQUE INDEX):

    Garante que todos os valores na(s) coluna(s) indexada(s) sejam únicos (permite múltiplos NULLs, a menos que a coluna seja NOT NULL).

  • Índices em Colunas Virtuais/Geradas (MySQL 5.7+):

    Permite indexar o resultado de uma expressão calculada a partir de outras colunas, armazenada como uma coluna virtual (STORED ou VIRTUAL).

  • Índices Funcionais (MySQL 8.0.13+):

    Permite criar um índice sobre o resultado de uma função. Ex: CREATE INDEX idx_name_lower ON t1((LOWER(name))); para buscas case-insensitive.

  • Índice FULLTEXT:

    Para buscas de texto completo (palavras-chave) em colunas CHAR, VARCHAR, ou TEXT. Usa algoritmos diferentes dos B-Trees.
