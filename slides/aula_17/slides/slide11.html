Estatísticas de Banco de Dados - Tópicos Importantes (Parte 2)

Algumas das estatísticas mais críticas que o otimizador do MySQL utiliza:

  • Cardinalidade da Tabela (Table Cardinality / TABLE_ROWS):

    O número total estimado de linhas na tabela. Ajuda a decidir se um Full Table Scan é viável ou se um índice é preferível.

  • Cardinalidade da Coluna/Índice (Column/Index Cardinality):

    O número estimado de valores distintos em uma coluna ou nas colunas de um índice. É crucial para avaliar a seletividade de um filtro ou índice. Um índice com alta cardinalidade (muitos valores únicos) é geralmente mais seletivo e útil.

  • Distribuição de Valores (Value Distribution / Histogramas):

    Como os valores estão distribuídos dentro de uma coluna. Ex: A coluna 'status_pedido' tem 70% 'CONCLUÍDO', 20% 'PENDENTE', 10% 'CANCELADO'. O MySQL (especialmente InnoDB) pode coletar histogramas (desde 5.7 para colunas específicas, mais robusto no 8.0) para melhorar as estimativas de cardinalidade para predicados de intervalo e valores não uniformemente distribuídos.

  • Número de Páginas/Blocos de Dados (Data Pages):

    Quantos blocos de disco (geralmente 16KB para InnoDB) a tabela ou índice ocupa. Relacionado ao custo de I/O para varreduras.

  • Presença e Frequência de Valores NULOs:

    Pode influenciar como os índices são usados e as estimativas para filtros como IS NULL ou IS NOT NULL.
