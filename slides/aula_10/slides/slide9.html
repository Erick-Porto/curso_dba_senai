INSERT: Lidando com Valores Especiais

  • NULL: Para colunas que permitem valores nulos, você pode inserir explicitamente NULL. 
    INSERT INTO Produtos (NomeProduto, CategoriaID, Preco, Descricao) VALUES ('Produto X', 1, 99.90, NULL);
  • DEFAULT: Para colunas que têm um valor padrão definido no CREATE TABLE (ex: Status VARCHAR(10) DEFAULT 'Ativo'), você pode usar a palavra-chave DEFAULT ou simplesmente omitir a coluna da lista (se estiver especificando colunas). 
    INSERT INTO Pedidos (ClienteID, DataPedido, StatusPedido) VALUES (10, NOW(), DEFAULT);
  • AUTO_INCREMENT / SERIAL (Chaves Primárias): Para colunas de ID que geram valores automaticamente, você geralmente:
    • Omite a coluna da lista de colunas no INSERT.
    • Ou passa DEFAULT ou NULL (depende do SGBD e configuração). O SGBD cuidará de gerar o próximo ID.
    -- ClienteID é AUTO_INCREMENT/SERIAL
INSERT INTO Clientes (Nome, Email) VALUES ('Novo Cliente', 'novo@email.com'); -- O ID será gerado
