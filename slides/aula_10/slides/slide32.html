Boas Práticas DML - Reforço Essencial (1/2)

Como um futuro DBA, a execução segura de DML é primordial:

  • 1. VERIFICAR ANTES DE AGIR: SEMPRE execute um SELECT * FROM ... WHERE ... com a MESMA condição WHERE que você usará no UPDATE ou DELETE. Confirme que SÓ as linhas desejadas são selecionadas.
    (Olhe antes de pular!)
  • 2. SEJA ESPECÍFICO NO WHERE: Evite condições vagas. Use Chaves Primárias (ex: WHERE ClienteID = 123) sempre que quiser alterar/deletar uma única linha específica. Para múltiplas linhas, garanta que a condição seja precisa.
    (Mire com precisão!)
  • 3. USE TRANSAÇÕES: Para qualquer operação DML que envolva múltiplos passos interdependentes (ex: criar pedido + itens) ou que seja crítica, envolva-a em uma transação (BEGIN...COMMIT/ROLLBACK).
    (Tudo ou nada!)
