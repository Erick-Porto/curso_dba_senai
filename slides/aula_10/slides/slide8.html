INSERT: Sintaxe Detalhada 2 (Omitindo Colunas)

Se você omitir a lista de colunas, DEVE fornecer valores para TODAS as colunas da tabela, exatamente na ORDEM em que elas foram definidas no CREATE TABLE.


-- Supondo que a tabela Clientes tenha (ClienteID, Nome, Email, Cidade, DataCadastro) NESTA ORDEM
INSERT INTO Clientes
VALUES (DEFAULT, 'Fernanda Souza', 'fe.souza@mail.org', 'Belo Horizonte', '2024-08-16');
-- 'DEFAULT' pode ser usado para colunas com auto-incremento ou valor padrão

Desvantagens / Riscos:

  • Menos legível.
  • Quebra facilmente! Se alguém alterar a ordem das colunas na tabela ou adicionar uma nova coluna no meio, este INSERT falhará ou inserirá dados nas colunas erradas.
  • Exige fornecer valor (ou NULL/DEFAULT) para *todas* as colunas.

Recomendação: Evite esta forma, prefira especificar as colunas.
