Modelagem Física: Mapeando Tipos para MySQL

Revisitando os tipos de dados, com foco na sintaxe e opções comuns do MySQL:

  • Inteiros: TINYINT (-128 a 127 ou 0 a 255 UNSIGNED), SMALLINT, MEDIUMINT, INT (ou INTEGER), BIGINT. Use UNSIGNED se o valor nunca for negativo (economiza um bit, dobra o range positivo).
    ClienteID INT UNSIGNED AUTO_INCREMENT PRIMARY KEY
  • Texto:
    • VARCHAR(n): 0 a 65,535 caracteres (limite real depende do charset e outras colunas).
    • CHAR(n): 0 a 255 caracteres.
    • TINYTEXT (até 255 bytes), TEXT (até 64KB), MEDIUMTEXT (até 16MB), LONGTEXT (até 4GB).
    Nome VARCHAR(100) NOT NULL, Descricao TEXT
  • Booleanos: BOOLEAN ou BOOL são sinônimos para TINYINT(1). (0 para Falso, 1 para Verdadeiro).
    Ativo BOOLEAN DEFAULT TRUE
  • Data e Hora:
    • DATE: 'YYYY-MM-DD'.
    • DATETIME: 'YYYY-MM-DD HH:MM:SS'. Range maior.
    • TIMESTAMP: 'YYYY-MM-DD HH:MM:SS'. Armazenado como segundos desde a época Unix (UTC), convertido para/de fuso horário da sessão. Útil para "criado em", "modificado em". Pode ter DEFAULT CURRENT_TIMESTAMP e ON UPDATE CURRENT_TIMESTAMP.
    DataCadastro TIMESTAMP DEFAULT CURRENT_TIMESTAMP
  • Decimal Exato: DECIMAL(P,S) ou NUMERIC(P,S). P = precisão total (máx 65), S = casas decimais (máx 30).
    Preco DECIMAL(10,2) NOT NULL
