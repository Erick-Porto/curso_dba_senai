Introdução à Modelagem Física: Do Papel ao SGBD Real

A Modelagem Física é a fase final, onde traduzimos o Modelo Lógico (independente de SGBD) para uma implementação concreta em um Sistema Gerenciador de Banco de Dados (SGBD) específico (ex: MySQL, PostgreSQL, SQL Server, Oracle).

Principais Considerações na Modelagem Física:

  • Escolha do SGBD: Cada SGBD tem suas próprias características, tipos de dados específicos, sintaxe DDL ligeiramente diferente, e mecanismos de otimização.
    • Para o nosso projeto, vamos focar no MySQL.
  • Mapeamento Exato de Tipos de Dados: Converter tipos lógicos (Texto, Número, Data) para os tipos específicos do SGBD (VARCHAR, INT, DATE, DECIMAL, etc., com seus tamanhos e precisões).
  • Definição de Índices: Além dos índices criados automaticamente por PKs e UNIQUEs, decidir quais outros índices criar para otimizar consultas.
  • Particionamento de Tabelas (Avançado): Para tabelas muito grandes, como dividi-las fisicamente.
  • Configurações de Storage Engine (MySQL): Ex: InnoDB (padrão, suporta transações, FKs), MyISAM (mais antigo, menos funcionalidades).
  • Geração dos Scripts SQL DDL: Escrever os comandos CREATE TABLE, ALTER TABLE, etc.
