MER - Refinando o Modelo

Alguns pontos para refinar nosso DER inicial:

  • Atributos Multivalorados:
    • Exemplo: Se um livro pudesse ter múltiplas "palavras-chave" diretamente como atributo.
    • Solução Correta (1FN): Criar uma nova entidade fraca ou tabela associativa. Ex: LIVRO_PALAVRASCHAVE (LivroID, PalavraChave).
    • No nosso caso, GENERO foi corretamente modelado como uma entidade separada com relacionamento N:M com LIVRO, o que já resolve isso.
  • Especialização/Generalização (Opcional para este projeto):
    • Se tivéssemos atributos comuns e específicos para diferentes tipos de "Pessoas" no sistema (ex: Autores, Membros, Funcionários).
    • Poderíamos ter uma superclasse PESSOA (com Nome, Documento) e subclasses AUTOR (com Biografia), MEMBRO (com DataValidadeCartao).
    • Prós: Reduz redundância de atributos comuns. Contras: Adiciona complexidade.
    • Para o nosso projeto de biblioteca, manter AUTOR e MEMBRO como entidades separadas é provavelmente mais simples e adequado ao escopo, a menos que haja muitos atributos em comum.
