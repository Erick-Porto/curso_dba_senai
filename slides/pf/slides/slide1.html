Trabalho Final da Disciplina: Desenvolvimento de BD (Parte 1/3)

Prezados(as) alunos(as), o trabalho final consistirá no processo completo de desenvolvimento de um banco de dados.

O tema do projeto é livre. Os critérios de avaliação são uniformes e não sofrerão alterações.

O trabalho será dividido em cinco fases principais. Vamos detalhar as duas primeiras:

1. Modelagem Conceitual

  • Deverá haver, no mínimo, 6 entidades.
  • Todos os relacionamentos entre as entidades deverão estar estabelecidos adequadamente, com suas respectivas cardinalidades.
  • O modelo conceitual deverá representar fielmente a lógica de funcionamento do sistema ou negócio proposto.

2. Modelagem Lógica

  • Todos os atributos referentes a cada entidade deverão ser especificados.
  • Os tipos de dados para cada atributo devem ser estabelecidos de forma a preservar a integridade e a lógica do projeto.
  • Deverão ser definidas restrições (constraints) apropriadas em cada tabela (ex: chaves primárias, chaves estrangeiras, not null, unique, check).
  • A cardinalidade dos relacionamentos deverá estar corretamente representada.
  • A normalização (pelo menos até a 3ª Forma Normal - 3FN) deverá ser aplicada, justificando a criação de tabelas auxiliares, se necessário.
